Foundation Component

Modals & Dialogs

Modal patterns for confirmations, forms, alerts, and focused interactions. Interrupt the user flow only when necessary.

01 — Philosophy

Intentional Interruption

Modals interrupt the user to focus attention on a specific task or decision. Use sparingly.

Context Switching

Only use modals when context switching is truly necessary—not for inline content that could stay in flow.

Clear Exit Path

Every modal needs a clear way out: close button, cancel action, or backdrop click.

04 — Modal Types

Alert Modal

Simple message with single action. Icon optional, centered layout.

Payment Complete

Your payment of $127.50 has been processed successfully.

Specifications

  • Size: Small (400px)
  • Icon: Optional, centered above title (40-48px)
  • Title: Centered, 18px, font-weight 600
  • Message: Centered, 14px, warm-600
  • Actions: Single centered button

Confirmation Modal

Decision requiring explicit choice. Shows consequences clearly.

Delete Transaction?

This will permanently delete transaction #TXN-2024-0847. This action cannot be undone.

Specifications

  • Size: Small (400px)
  • Title: Left-aligned, 18px
  • Message: Description of consequences
  • Actions: Ghost cancel + Primary/Destructive confirm

Form Modal

Input collection in modal context. Inline validation, submit disabled until valid.

Add New Program

Connect a loyalty program to your wallet

Account number is required

Specifications

  • Size: Default (500px) or Large (640px)
  • Body: Form fields with standard input styling
  • Actions: Ghost cancel + Primary submit
  • Validation: Inline errors, submit disabled until valid
  • Error state: magenta-500 border + error text

Information Modal

Content display without required action. Rich content support.

How Points Work

Points are converted to dollars at checkout. The exchange rate varies by program:

  • Target Circle: 100 points = $1.00
  • AAdvantage: 100 miles = $1.50
  • Hilton Honors: 200 points = $1.00

Specifications

  • Size: Default or Large
  • Body: Rich content (text, images, lists)
  • Actions: Single "Close" or "Got it" button

08 — Animation

Opening

  1. Backdrop fades in (150ms, ease-out)
  2. Modal scales up from 95% and fades in (200ms, ease-out)

Closing

  1. Modal scales down to 95% and fades out (150ms, ease-in)
  2. Backdrop fades out (150ms, ease-in)
@keyframes modal-enter {
  from {
    opacity: 0;
    transform: scale(0.95);
  }
  to {
    opacity: 1;
    transform: scale(1);
  }
}

@keyframes modal-exit {
  from {
    opacity: 1;
    transform: scale(1);
  }
  to {
    opacity: 0;
    transform: scale(0.95);
  }
}

10 — Focus & Accessibility

Focus Management

  • 1 On open: Focus moves to first focusable element (or close button)
  • 2 Tab trapping: Tab cycles through modal elements only
  • 3 On close: Focus returns to element that triggered modal
  • 4 Escape key: Closes modal (unless during critical operation)

ARIA Requirements

  • role="dialog" on container
  • aria-modal="true" on container
  • aria-labelledby referencing title ID
  • aria-describedby for description (optional)
  • Modal announced when opened
  • Content readable in logical order

Backdrop Behavior

Click to close

Default behavior: clicking backdrop closes modal. Can be disabled for important forms.

Visual state

Backdrop dims page content. Page content should not be interactive while modal is open.

13 — Implementation Checklist

Container

  • Border radius: 12px
  • Shadow: 0 25px 50px -12px rgba(0,0,0,0.25)
  • Max height: 90vh
  • Sizes: 400px, 500px, 640px, 800px, 90vw

Backdrop

  • Color: intaglio-900/60 (light) / intaglio-950/80 (dark)
  • Click to close (default behavior)

Header

  • Padding: 20px 24px
  • Title: 18px, font-weight 600
  • Close button: top-right corner

Footer

  • Padding: 16px 24px
  • Border-top: 1px warm-200 (light) / intaglio-700 (dark)
  • Buttons: right-aligned, 12px gap

Animation

  • Enter: 200ms scale + fade
  • Exit: 150ms scale + fade

Accessibility

  • Focus trap within modal
  • ESC closes modal
  • aria-modal="true"
  • Focus returns to trigger on close
