Core Foundation

Typography

Font families, type scale, and usage guidelines for product interfaces.

Font Families

Fraunces

Variable serif with optical sizing. 9 weights (100-900), roman and italic. Warm character with playful personality that flexes from authoritative to approachable.

font-display

Fallback: Georgia, serif

Inter

Primary UI font. Excellent legibility at all sizes. Use for body, labels, and navigation.

font-body or font-sans

Fallback: system-ui, sans-serif

Outfit

Brand font. Reserved for logo lockups and special brand moments only.

font-brand

Fallback: system-ui, sans-serif

Usage Guidelines

Fraunces

  • ✓ Page titles and hero headlines
  • ✓ Section headers (H1, H2)
  • ✓ Marketing display text (300-400 weight)
  • ✓ Pull quotes and testimonials (italic)
  • ✗ Never for body text or UI labels
  • ✗ Never for buttons or navigation

Inter

  • ✓ All body text and paragraphs
  • ✓ Buttons, labels, form fields
  • ✓ Navigation and menus
  • ✓ H3, H4 in product contexts
  • ✓ Tables, lists, metadata
  • ✓ Captions and helper text

Weight Usage

  • Regular (400) — Body text, descriptions
  • Medium (500) — Labels, subtle emphasis
  • Semibold (600) — Buttons, headings, strong emphasis
  • Bold (700) — Rarely used; very strong emphasis only
  • Light (300) — Large display text only, never for body

Fraunces Weight Scale

Fraunces is a variable font with 9 weights. Use lighter weights for large display text, heavier weights for emphasis and authority.

Fraunces 100-200 Ultra-light, use sparingly for oversized display
Fraunces 300-400 Light to regular, shopper marketing, friendly tone
Fraunces 500-600 Medium, standard headlines, balanced authority
Fraunces 700-800 Bold, merchant emphasis, strong statements
Fraunces 900 Black, use sparingly for maximum impact

Serial Label Style

Currency-inspired uppercase labels. Used for section markers and category labels.

Serial labels are NOT interactive elements. They use indigo to create visual emphasis without implying clickability. Emerald is reserved strictly for interactive elements (buttons, links). This separation ensures users can immediately distinguish navigational affordances from organizational markers.

Primary Serial

indigo-600

Secondary Serial

warm-500

Serial on Dark

indigo-300

When to use Primary (indigo)

  • • Section headers introducing main content
  • • Category labels in documentation
  • • Feature area markers

When to use Secondary (warm)

  • • Supporting metadata labels
  • • Timestamps and attribution
  • • Lower-priority organizational markers
