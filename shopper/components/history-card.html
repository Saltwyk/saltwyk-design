Shopper Component

History Card

Celebratory cards showing completed purchases where shoppers successfully used their points.

Preview Mode

| Full Page

Overview

The History Card shows completed transactions where shoppers used their accumulated points. Cards are intentionally simple — just the product and which merchants contributed. Detailed breakdowns live in the Transaction Detail Modal, which serves as the hero element.

Purpose

Show completed purchases, list contributing merchants

Location

Passport "Snagged" section, transaction history

Thread Rule

One thread per screen — only on the modal hero

Design Decision: One Thread Per Screen

The Fresh thread accent (Cyan → Lime) appears only on the Transaction Detail Modal — the hero element of the history view. Individual cards stay simple and thread-free to maintain visual hierarchy.

  • • Cards show what was purchased and which merchants contributed
  • • The modal shows the detailed breakdown with per-merchant line items
  • • Thread-bearing elements have no rounded top corners

Usage Guidelines

Do

  • • Keep cards simple — product, retailer, merchant names
  • • Reserve thread for the modal (hero) only
  • • Include per-merchant reference IDs on receipt
  • • Provide support contact for each merchant
  • • Remove top corners on thread-bearing elements

Don't

  • • Don't put thread on cards (one thread per screen)
  • • Don't show points breakdown on cards
  • • Don't use credit card programs as examples
  • • Don't round top corners with thread present
  • • Don't omit support metadata on receipts
