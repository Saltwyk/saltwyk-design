1. Philosophy

  • Dropdowns reveal options on demand, keeping interfaces clean
  • They should feel responsive and lightweight — appear instantly, easy to dismiss
  • Consistent trigger patterns help users recognize interactive elements

2. Dropdown Types Overview

Different dropdown types serve different interaction patterns.

Type Trigger Use Case
Select Menu Input-style trigger Form field selection
Action Menu Button trigger Actions on an item
Navigation Menu Nav item or button Site/app navigation

12. Keyboard Navigation & Accessibility

Keyboard interactions and ARIA requirements.

Keyboard Navigation

  • Enter / Space — Open menu, select item
  • Escape — Close menu
  • ↑ / ↓ — Navigate items
  • Home / End — First / last item
  • Type characters — Jump to matching item
  • Tab — Close menu, move focus

ARIA Requirements

  • Trigger: aria-haspopup, aria-expanded
  • Select menu: role="listbox"
  • Action menu: role="menu"
  • Items: role="option" or role="menuitem"
  • Selected: aria-selected="true"
  • Disabled: aria-disabled="true"

13. Code Reference

CSS implementations using Tailwind utility classes.

Menu Container

/* Light mode */ .dropdown-menu { @apply absolute z-50; @apply bg-white border border-warm-200 rounded-lg; @apply py-1 min-w-[180px] max-w-[320px]; @apply overflow-auto max-h-[320px]; box-shadow: 0 4px 16px rgba(0,0,0,0.12); } /* Dark mode */ .dropdown-menu-dark { @apply bg-intaglio-800 border-intaglio-700; box-shadow: 0 4px 16px rgba(0,0,0,0.3); }

Menu Item

/* Light mode */ .dropdown-item { @apply flex items-center gap-2 px-3 py-2; @apply text-sm text-ink cursor-pointer; @apply transition-colors duration-100; } .dropdown-item:hover { @apply bg-warm-50; } .dropdown-item-selected { @apply bg-emerald-50 text-emerald-700 font-medium; } .dropdown-item-disabled { @apply text-warm-400 cursor-not-allowed; } .dropdown-item-destructive { @apply text-magenta-600; } .dropdown-item-destructive:hover { @apply bg-magenta-50; }

Select Trigger

.select-trigger { @apply flex items-center justify-between; @apply w-full px-3 py-2.5; @apply bg-white border-[1.5px] border-warm-300 rounded-md; @apply text-sm text-ink; @apply transition-colors duration-150; } .select-trigger:hover { @apply border-warm-400; } .select-trigger[data-state="open"] { @apply border-emerald-500 ring-2 ring-emerald-500/20; } .select-trigger-icon { @apply w-4 h-4 text-warm-400 transition-transform; } .select-trigger[data-state="open"] .select-trigger-icon { @apply rotate-180; }

Animation

@keyframes dropdown-enter { from { opacity: 0; transform: scale(0.95) translateY(-4px); } to { opacity: 1; transform: scale(1) translateY(0); } } @keyframes dropdown-exit { from { opacity: 1; transform: scale(1); } to { opacity: 0; transform: scale(0.95); } } .dropdown-enter { animation: dropdown-enter 150ms ease-out; transform-origin: top; } .dropdown-exit { animation: dropdown-exit 100ms ease-in forwards; }

14. Implementation Checklist

Key specifications to verify during implementation.

Menu Container

  • ☐ Border radius: 8px
  • ☐ Shadow: 0 4px 16px rgba(0,0,0,0.12)
  • ☐ Border: warm-200 (light) / intaglio-700 (dark)
  • ☐ Max height: 320px with scroll
  • ☐ Padding: 4px top/bottom

Menu Item

  • ☐ Height: 36px default
  • ☐ Padding: 8px 12px
  • ☐ Hover: warm-50 (light) / intaglio-700 (dark)
  • ☐ Selected: emerald-50 + emerald-700 text (light)
  • ☐ Selected dark: emerald-500/15 + emerald-400 text

Animation

  • ☐ Enter: 150ms scale + fade
  • ☐ Exit: 100ms scale + fade
  • ☐ Transform origin based on position

Positioning

  • ☐ Flip when insufficient space
  • ☐ Stay within viewport
  • ☐ 8px minimum from edges
