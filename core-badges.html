Component Specification

Badges & Status Indicators

Implementation specification for badges, status indicators, count badges, and tags. Every variant, semantic color, and pixel value defined.

1. Philosophy

Badges communicate status, category, or count at a glance. They should be immediately scannable without reading—color carries meaning.

Use consistent semantic colors across the product. Success is always emerald, errors always magenta. This consistency builds user confidence and speeds comprehension.

4. Semantic Color Mapping

Colors must be used consistently across the product. This mapping is non-negotiable.

Semantic Color Family Use Cases Preview
Success Emerald Completed, active, connected, approved, paid, cleared Completed
Warning Orange Pending, expiring soon, needs attention, processing Pending
Error Magenta Failed, declined, error, disconnected, overdue Failed
Info Cyan New, updated, information, in progress New
Neutral Warm Gray Default, inactive, archived, draft Draft

Never use:

  • • Success (emerald) for pending states
  • • Error (magenta) for warnings
  • • Random colors for visual variety
  • • Info (cyan) where success (emerald) is appropriate

13. Usage Guidelines

Do

  • • Use consistent colors for consistent meanings
  • • Use filled badges for primary status
  • • Use outline badges for secondary/subtle status
  • • Use dots for minimal footprint
  • • Keep badge text short (1-2 words)
  • • Use sentence case for badge labels

Don't

  • • Mix color meanings across the product
  • • Use too many badges on one element
  • • Use bright badges everywhere (loses impact)
  • • Use badges for actions (use buttons instead)
  • • Use ALL CAPS in badge labels
  • • Create custom colors outside the semantic palette

14. Implementation Checklist

Verify your badge implementation covers all requirements.

Base Badge

  • Border radius: 9999px (pill)
  • Font weight: 500
  • Default size: 24px height, 8px horizontal padding
  • Font size: 12px

Semantic Colors (Light)

  • Success: emerald-100 bg, emerald-700 text
  • Warning: orange-100 bg, orange-700 text
  • Error: magenta-100 bg, magenta-700 text
  • Info: cyan-100 bg, cyan-700 text
  • Neutral: warm-100 bg, warm-700 text

Semantic Colors (Dark)

  • Success: emerald-500/20 bg, emerald-400 text
  • Warning: orange-500/20 bg, orange-400 text
  • Error: magenta-500/20 bg, magenta-400 text
  • Info: cyan-500/20 bg, cyan-400 text
  • Neutral: warm-500/20 bg, warm-300 text

Count Badge

  • Circle shape for 1-2 digits
  • Pill shape for 3+ digits
  • Truncate to 99+ or 999+
  • Position offset: -4px

Tags

  • Border radius: 6px (not pill)
  • Close button: right side, 14px
  • Hover state on close button

Dots

  • Sizes: 6px, 8px, 10px
  • Pulse animation available
