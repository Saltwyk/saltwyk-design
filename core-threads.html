Guidelines

Thread Usage Principles

One thread per page

A single thread creates focus. Multiple threads compete and dilute impact. Never use more than one thread on a page.

Top placement is standard

Thread lines at the top of heroes and containers create a strong entry point. Use squared top corners (rounded-b-lg) to allow full-bleed threads.

Threads signal location, not decoration

Each thread tells the user where they are in the Saltwyk ecosystem:

  • Spectrum → Brand moment (Design System or Marketing)
  • Fresh → Shopper experience
  • Cool → Merchant experience
  • Ocean → Documentation

Threads require contrast

Threads must pop against their background. They are high-saturation, vibrant elements. Never use muted or desaturated thread colors — that defeats their purpose.

Restrictions

Color Usage Restrictions

Cyan is reserved for thread decorations only

Cyan exists solely for thread gradient accents. Never use cyan for UI elements, backgrounds, badges, or interactive states—use indigo for those purposes.

✓ Allowed

  • Thread gradients
  • Decorative spectrum accents

✗ Not Allowed

  • Info badges (use indigo)
  • Focus rings (use indigo)
  • Backgrounds
  • Interactive states

Hot colors (Orange, Magenta) in threads only

Orange and Magenta read as warning/error when used in isolation. They are safe within gradient threads where they blend with cooler colors, but should not anchor standalone UI elements.

✓ Safe Usage

  • Inside Spectrum thread
  • Inside Cool thread (magenta)
  • Inside Flame thread (both)

✗ Risky Usage

  • Solid orange backgrounds
  • Orange-dominant UI elements
  • Isolated magenta accents
