Component Specification

Tooltips

Implementation specification for Saltwyk tooltip components. Brief, contextual information on hover or focus.

1. Philosophy

Tooltips provide contextual information on hover or focus. They should be brief, helpful, and non-essential—the information should be discoverable elsewhere in the interface.

Tooltips must not block important content or require precise mouse positioning. They appear after a short delay to avoid visual noise during normal navigation.

7. Behavior

Timing

Show delay (hover) 300ms
Show delay (focus) 0ms (immediate)
Hide delay 100ms
Duration No auto-dismiss

Interactions

  • • Tooltip is not interactive by default
  • • Mouse can pass over tooltip to reach elements behind
  • • For interactive tooltips (with links), use Popover instead
  • • Hide delay allows moving mouse to tooltip if needed

8. Animation

Enter Animation

/* Enter: 100ms, ease-out */ @keyframes tooltip-enter { from { opacity: 0; transform: scale(0.95); } to { opacity: 1; transform: scale(1); } }

Duration: 100ms, ease-out easing

Exit Animation

/* Exit: 75ms, ease-in */ @keyframes tooltip-exit { from { opacity: 1; } to { opacity: 0; } }

Duration: 75ms, ease-in easing

9. Accessibility

Trigger Requirements

  • • Focusable element (button, link, input)
  • • aria-describedby pointing to tooltip id
  • • OR aria-label containing tooltip text

Tooltip Requirements

  • • role="tooltip"
  • • id matching trigger's aria-describedby

Keyboard Behavior

Focus Shows tooltip
Blur Hides tooltip
ESC Dismisses tooltip

Screen Readers

  • • Tooltip content announced when trigger focused
  • • Content should be supplementary, not essential

ARIA Structure Example

<!-- Trigger --> <button aria-describedby="tooltip-1"> Save </button> <!-- Tooltip --> <div role="tooltip" id="tooltip-1"> Save changes (⌘S) </div>

11. Code Reference

Tooltip Container

.tooltip { @apply absolute z-50; @apply px-2.5 py-1.5 rounded-md; @apply bg-intaglio-900 text-white; @apply text-xs leading-snug; @apply max-w-[240px]; @apply pointer-events-none; } .tooltip-dark { @apply bg-intaglio-700; }

Tooltip Arrow

.tooltip-arrow { @apply absolute w-0 h-0; border: 6px solid transparent; } /* Top placement - arrow points down */ .tooltip[data-side="top"] .tooltip-arrow { @apply -bottom-3 left-1/2 -translate-x-1/2; border-top-color: theme('colors.intaglio.900'); border-bottom: none; } /* Bottom placement - arrow points up */ .tooltip[data-side="bottom"] .tooltip-arrow { @apply -top-3 left-1/2 -translate-x-1/2; border-bottom-color: theme('colors.intaglio.900'); border-top: none; }

Tooltip Variants

/* Title + Description */ .tooltip-title { @apply font-semibold text-white; } .tooltip-description { @apply text-warm-300 mt-1; } /* Keyboard Shortcut */ .tooltip-shortcut { @apply text-warm-400 font-mono ml-2; }

Animation

.tooltip-enter { animation: tooltip-enter 100ms ease-out; } .tooltip-exit { animation: tooltip-exit 75ms ease-in forwards; } @keyframes tooltip-enter { from { opacity: 0; transform: scale(0.95); } to { opacity: 1; transform: scale(1); } } @keyframes tooltip-exit { from { opacity: 1; } to { opacity: 0; } }

12. Implementation Checklist

Container

  • Background: intaglio-900 (light) / intaglio-700 (dark)
  • Text: white, 12px
  • Padding: 6px 10px
  • Border radius: 6px
  • Max width: 240px

Arrow

  • Size: 6px
  • Matches background color
  • Positioned based on placement

Behavior

  • Show delay: 300ms (hover), 0ms (focus)
  • Hide delay: 100ms
  • Auto-positioning to stay in viewport

Animation

  • Enter: 100ms scale + fade
  • Exit: 75ms fade

Accessibility

  • role="tooltip"
  • aria-describedby on trigger
  • Shows on focus
  • ESC dismisses
