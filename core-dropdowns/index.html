1. Philosophy

  • Dropdowns reveal options on demand, keeping interfaces clean
  • They should feel responsive and lightweight — appear instantly, easy to dismiss
  • Consistent trigger patterns help users recognize interactive elements

6. Keyboard Navigation & Accessibility

Keyboard interactions and ARIA requirements.

Keyboard Navigation

  • Enter / Space — Open menu, select item
  • Escape — Close menu
  • ↑ / ↓ — Navigate items
  • Home / End — First / last item
  • Type characters — Jump to matching item
  • Tab — Close menu, move focus

ARIA Requirements

  • Trigger: aria-haspopup, aria-expanded
  • Select menu: role="listbox"
  • Action menu: role="menu"
  • Items: role="option" or role="menuitem"
  • Selected: aria-selected="true"
  • Disabled: aria-disabled="true"

7. Implementation Checklist

Key specifications to verify during implementation.

Menu Container

  • ☐ Border radius: 8px
  • ☐ Shadow: 0 4px 16px rgba(0,0,0,0.12)
  • ☐ Border: warm-200 (light) / intaglio-700 (dark)
  • ☐ Max height: 320px with scroll
  • ☐ Padding: 4px top/bottom

Menu Item

  • ☐ Height: 36px default
  • ☐ Padding: 8px 12px
  • ☐ Hover: warm-50 (light) / intaglio-700 (dark)
  • ☐ Selected: indigo-50 + indigo-700 text (light)
  • ☐ Selected dark: indigo-500/15 + indigo-400 text

Animation

  • ☐ Enter: 150ms scale + fade
  • ☐ Exit: 100ms scale + fade
  • ☐ Transform origin based on position

Positioning

  • ☐ Flip when insufficient space
  • ☐ Stay within viewport
  • ☐ 8px minimum from edges
