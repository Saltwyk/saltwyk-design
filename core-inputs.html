Component Specification

Inputs & Form Fields

Implementation specification for Saltwyk form components. Every state, variant, and pixel value defined.

1. Philosophy

Form inputs are where users provide information—they must feel stable and trustworthy. Clear focus states guide users through forms, providing confidence about where they are.

Validation feedback is immediate and helpful, never punishing. Error states are clear but not alarming—they guide users toward correction without creating anxiety.

Fire Opal Character

Inputs are darker wells (intaglio-800) sitting in the dark surface (intaglio-900). Emerald focus state provides the "thread burning through" effect. Good contrast between input background and warm-100 text.

18. Implementation Checklist

Verification checklist for developers implementing form components.

Text Input

  • ☐ Border radius: 6px (rounded-md)
  • ☐ Border width: 1.5px
  • ☐ Default border: warm-300 (light) / intaglio-600 (dark)
  • ☐ Focus border: emerald-500
  • ☐ Error border: magenta-500
  • ☐ Focus ring: 2px emerald-500, 2px offset
  • ☐ Placeholder: warm-400 (light) / warm-500 (dark)

Checkbox

  • ☐ Size: 18px 18px
  • ☐ Border radius: 4px
  • ☐ Checked background: emerald-500
  • ☐ Check icon: white, 12px
  • ☐ Label gap: 8px

Radio

  • ☐ Size: 18px 18px
  • ☐ Border radius: 50%
  • ☐ Selected dot: emerald-500, 8px
  • ☐ Group gap: 12px between items

Toggle

  • ☐ Track: 44px 24px
  • ☐ Thumb: 20px circle
  • ☐ On color: emerald-500
  • ☐ Animation: 150ms ease-out

Dark Mode

  • ☐ Input background: intaglio-800
  • ☐ Input border: intaglio-600
  • ☐ Text color: warm-100
  • ☐ Placeholder: warm-500
  • ☐ Focus ring offset: intaglio-900

Accessibility

  • ☐ All inputs have associated labels
  • ☐ Error messages linked via aria-describedby
  • ☐ Focus visible on all interactive elements
  • ☐ Disabled inputs have aria-disabled="true"

19. Code Reference

Copy-pasteable CSS/Tailwind classes for each input variant.

Text Input (Light)

/* Base */
.input {
  @apply w-full px-3 py-2 bg-white border-[1.5px] border-warm-300 rounded-md;
  @apply text-sm text-ink placeholder:text-warm-400;
  @apply transition-all duration-150 ease-out;
  @apply focus:border-emerald-500 focus:ring-2 focus:ring-emerald-500 focus:ring-offset-2;
}

/* Hover */
.input:hover {
  @apply border-warm-400;
}

/* Disabled */
.input:disabled {
  @apply bg-warm-50 border-warm-200 text-warm-400 cursor-not-allowed;
}

/* Error */
.input-error {
  @apply border-magenta-500 focus:ring-magenta-500;
}

Text Input (Dark)

/* Base */
.input-dark {
  @apply w-full px-3 py-2 bg-intaglio-800 border-[1.5px] border-intaglio-600 rounded-md;
  @apply text-sm text-warm-100 placeholder:text-warm-500;
  @apply transition-all duration-150 ease-out;
  @apply focus:border-emerald-500 focus:ring-2 focus:ring-emerald-500 focus:ring-offset-2;
  @apply focus:ring-offset-intaglio-900;
}

/* Hover */
.input-dark:hover {
  @apply border-intaglio-500;
}

/* Disabled */
.input-dark:disabled {
  @apply bg-intaglio-900 border-intaglio-700 text-warm-600 cursor-not-allowed;
}

Checkbox

/* Base */
.checkbox {
  @apply w-[18px] h-[18px] rounded border-[1.5px] border-warm-300 bg-white;
  @apply transition-all duration-150 ease-out cursor-pointer;
  @apply checked:bg-emerald-500 checked:border-emerald-500;
  @apply focus:ring-2 focus:ring-emerald-500 focus:ring-offset-2;
}

/* Hover */
.checkbox:hover {
  @apply border-warm-400;
}

/* Checked Hover */
.checkbox:checked:hover {
  @apply bg-emerald-600 border-emerald-600;
}

Toggle

/* Track */
.toggle-track {
  @apply w-11 h-6 bg-warm-200 rounded-full relative cursor-pointer;
  @apply transition-colors duration-150;
}

.toggle-track[data-state="checked"] {
  @apply bg-emerald-500;
}

/* Thumb */
.toggle-thumb {
  @apply w-5 h-5 bg-white rounded-full shadow-sm;
  @apply absolute top-0.5 left-0.5;
  @apply transition-transform duration-150 ease-out;
}

.toggle-track[data-state="checked"] .toggle-thumb {
  @apply translate-x-5;
}
