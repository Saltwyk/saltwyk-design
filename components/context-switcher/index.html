Component Specification

Context Switcher

Implementation specification for switching between Shopper Passport and Merchant Dashboard contexts.

1. Philosophy

Context switchers help users navigate between distinct product areas. They must provide clear wayfinding—users should always know where they are. Transitions should feel intentional, not accidental.

Visual treatment must reinforce the context: Bright Opal (light) for Shopper Passport, Fire Opal (dark) for Merchant Dashboard.

2. Use Cases

Primary: Shopper ↔ Merchant

Users who are both shoppers and merchants need to switch between:

  • Shopper Passport — Personal rewards wallet, point balances, transaction history
  • Merchant Dashboard — Business operations, clearing balance, settlements

Secondary Use Cases

  • Multi-merchant access — Consultants/agencies managing multiple merchants
  • Organization switcher — Switching between different merchant accounts
  • Role switcher — Different permission levels within same organization

12. Implementation Checklist

Context Indicator

  • Shows current context name and type
  • Icon matches context (wallet/building)
  • Chevron indicates expandable
  • Hover state implemented
  • Light/dark variants

Switcher Menu

  • Lists available contexts
  • Current context marked with checkmark
  • Hover states on options
  • Section dividers if multiple types
  • Search if >5 options

Visual Reinforcement

  • Sidebar changes light/dark on switch
  • Icon colors match context type
  • URL reflects current context
  • Transition animation on switch

Accessibility

  • aria-haspopup and aria-expanded on trigger
  • role="menu" on dropdown
  • Keyboard navigation
  • Screen reader announcements
