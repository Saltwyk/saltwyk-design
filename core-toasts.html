Toast Notifications

Temporary, non-blocking feedback for user actions and system events.

Toast Anatomy

The structural components of a toast notification.

Payment successful

Status indicator (success, warning, error, info)

Title (optional) and message

Action

Optional button or link

Dismiss

Close button

Progress

Auto-dismiss countdown (optional)

Toast Types

Different types for various notification purposes.

Success

Transaction cleared
$127.50 has been processed successfully.

Use for completed actions, confirmations

Warning

Balance running low
Add funds to avoid transaction delays.

Use for attention-needed, non-critical

Error

Payment failed
Unable to process transaction. Please try again.

Use for failed actions, errors

Info

New feature available
You can now connect multiple loyalty programs.

Use for informational updates, tips

Toast vs Alert Banner

When to use toasts versus persistent alert banners.

Use Toast

  • • Temporary feedback (auto-dismiss)
  • • Non-blocking, non-critical
  • • Action confirmations
  • • Background process updates
  • • Can be missed without consequence

Use Alert Banner

  • • Persistent, requires acknowledgment
  • • Critical information
  • • System-wide announcements
  • • Errors requiring user action
  • • Must not be missed

Accessibility

ARIA attributes and screen reader considerations.

ARIA Attributes

  • • role="alert" for errors
  • • role="status" for success/info
  • • aria-live="polite" for info
  • • aria-live="assertive" for errors

Focus Management

  • • Toast should not steal focus
  • • Action buttons must be focusable
  • • Close button: aria-label="Dismiss"
  • • Announce to screen readers on appear
