Component Specification

Partnership Tier

Transaction Detail

Displays individual transaction information including product, currency amount, and multi-merchant point contributions. One of the primary mobile/desktop screens for merchants.

Preview Mode

Currency Display Context

This component (along with reports) is one of the only places in the merchant app showing actual currency amounts. Points are the primary value display elsewhere.

Internal-Only Status States

Status badges (completed, pending, failed) are only visible to internal Saltwyk team members. Merchants see transactions without status indicators.

6. Design Decisions

Why show each contribution's UUID?

Each merchant contribution is a separate ledger entry with its own UUID. This enables precise debugging when merchants have questions about specific point deductions and provides an audit trail for each contribution.

Why include wishlist metadata?

Merchants benefit from understanding shopper intent. Knowing an item was on a wishlist for 47 days shows deliberate purchase planning vs. impulse buying. This data informs merchant decisions about inventory, pricing, and point incentives.

Why hide status from merchants?

Merchants don't need to see pending/failed states — they only care about completed transactions. Status visibility adds complexity without value. Internal teams need status for operations; merchants just need their transaction history.

Why use logo.dev instead of stored logos?

Logo.dev provides always-current brand logos without storage/maintenance overhead. Logos update automatically when brands rebrand. The API token ensures proper licensing and CDN-backed performance.

7. Usage Guidelines

Do

  • • Show product name and retailer prominently
  • • Display all contributing merchants with logos
  • • Include individual contribution UUIDs
  • • Show wishlist metadata when available
  • • Use tabular-nums for all currency/point values
  • • Provide copy buttons for all UUIDs

Don't

  • • Show status badges to merchants
  • • Truncate contribution UUIDs in detail view
  • • Use placeholder logos when logo.dev fails (show letter fallback)
  • • Hide multi-merchant attribution
  • • Round or truncate currency amounts
  • • Omit wishlist metadata when available
