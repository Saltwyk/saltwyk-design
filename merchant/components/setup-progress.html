Setup Progress

A compact progress indicator used in merchant onboarding flows. Shows completion status through dots and connecting lines, optimized for multi-step setup wizards.

6. Design Decisions

Why 10px dots instead of 32px step indicators?

Merchant onboarding often has 5-7 steps. At 32px per indicator, the stepper would dominate the mobile header. The 10px dot pattern stays compact while still clearly communicating progress. For desktop layouts requiring more detail, use the labeled or vertical variants.

Why indigo for active (not emerald border)?

The core stepper uses emerald border for "current" state. Setup Progress uses indigo-500 with a ring to better distinguish the active step from completed steps at small sizes. The ring adds visual weight and draws focus to the current position.

When to use each variant?

Inline Dots: Mobile headers, compact spaces. Labeled: Desktop forms where step names add clarity. Vertical: Sidebars, summary views, or when step descriptions are needed.

7. Usage Guidelines

Do

  • • Use for linear, sequential flows (onboarding, checkout)
  • • Show all steps upfront so users know the full scope
  • • Allow navigation back to completed steps
  • • Pair with a step card showing current step details
  • • Use error state when a step requires attention

Don't

  • • Use for non-linear or branching flows
  • • Show more than 7 steps (consider grouping)
  • • Use without additional context (add step card or labels)
  • • Mix with other progress indicators on the same screen
  • • Skip steps or allow jumping ahead to incomplete ones
