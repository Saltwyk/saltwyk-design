Component Specification

Money Flow Diagram

Static educational illustrations explaining how money flows through Saltwyk's system. Used in onboarding Phase 3 (Banking) and Phase 5 (Clearing Balance).

1. Overview

Purpose

Visually explain how money flows from merchant funding through shopper transactions to settlements.

Format

Static SVG illustrations. No animations or interactivity for MVP.

Usage

Onboarding Phase 3 & 5, Settings help sections, documentation.

2. Variant A: Dual-Purpose Bank Account

Shows how one bank account serves two purposes: funding the clearing balance and receiving settlements. Use in Phase 3 (Banking) to explain why we need bank account info.

3. Variant B: Full Transaction Flow

Shows the complete lifecycle from funding through shopper purchase to settlement. Use in Phase 5 (Clearing Balance) before asking for initial funding.

4. When to Use Each Variant

A

Dual-Purpose Bank Account

  • Phase 3 (Banking): Before bank account input
  • Question answered: "Why do you need my bank account?"
  • Focus: One account serves funding + settlements
  • Settings: Finance help section
B

Full Transaction Flow

  • Phase 5 (Clearing Balance): Before funding calculator
  • Question answered: "What is a clearing balance?"
  • Focus: Complete money lifecycle
  • Settings: "How it works" explanations

6. Accessibility

Required Elements

  • • role="img" on figure element
  • • aria-labelledby pointing to description
  • • Screen reader description in figcaption
  • • Expandable text alternative below diagram
  • • Sufficient color contrast (4.5:1 minimum)

Implementation Pattern

<figure role="img" aria-labelledby="desc">
  <figcaption id="desc" class="sr-only">
    Description for screen readers...
  </figcaption>
  <svg>...</svg>
  <details>
    <summary>View text description</summary>
    <ol>...text alternative...</ol>
  </details>
</figure>

7. Responsive Behavior

Desktop (≥768px)

  • • Full 600px width
  • • 24px container padding
  • • All text at specified sizes

Tablet (640-768px)

  • • Scales proportionally
  • • SVG viewBox maintains aspect ratio
  • • Container adjusts to fit

Mobile (<640px)

  • • 100% width with 16px padding
  • • SVG scales down automatically
  • • Text remains readable

8. Design Decisions

Why static SVG (not animated)?

MVP simplicity, works everywhere without JS dependencies, easier to provide text alternatives, instant load with small file size.

Why two variants?

Different contexts need different focus. Dual-purpose answers "Why do you need my bank?" while full flow answers "What is clearing balance?"

Why emerald and indigo colors?

Emerald = merchant's action (funding). Indigo = system action (settlements). Clear visual distinction that matches Fire Opal design system.

Why include FDIC and Stripe badges?

Addresses merchant anxiety about fund safety. Builds trust through credibility signals and transparency about who holds their money.
