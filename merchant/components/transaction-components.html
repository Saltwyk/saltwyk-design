Transaction Components

Filter tabs, status badges, and data table patterns used on the Transactions screen.

Preview Mode

Overview

The Transactions screen displays all rewards redemptions made by shoppers at this merchant. Each row represents points converted to purchase value. Merchants use this to reconcile activity and track their Saltwyk performance.

Purpose

View, search, and filter redemption history

Location

Merchant Dashboard → Transactions

Data Density

High — optimized for scanning many rows quickly

Design Decisions

Emerald for Status, Not Decoration: On merchant screens, emerald green is reserved exclusively for actionable elements (buttons, active nav) and success states (completed badges). Thread gradients appear only in the page divider — never in data displays.

Status Color Semantics:

  • • Completed uses emerald to signal "money received"
  • • Pending uses warm orange to indicate "waiting for settlement"
  • • Failed uses magenta for clear error visibility

Table Row Styling: Rows stay neutral with no colored backgrounds that compete with status badges. Hover state uses subtle warm-50 background only.

Usage Guidelines

Do

  • • Use monospace font for transaction IDs
  • • Use tabular-nums for currency alignment
  • • Show count in filter tabs
  • • Keep status badges lowercase
  • • Provide clear empty states with next action

Don't

  • • Don't use colored row backgrounds
  • • Don't show thread gradients in table cells
  • • Don't use emerald for non-success states
  • • Don't truncate transaction IDs
  • • Don't hide filter counts
