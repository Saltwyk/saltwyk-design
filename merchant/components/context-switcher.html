Merchant Component

Context Switcher

Top-right dropdown for switching between merchant accounts, accessing shopper profile, settings, and sign out.

1. Overview

The context switcher provides account-level navigation for merchants. It allows users to switch between multiple merchant accounts (if they have access), return to their shopper profile, access settings, and sign out.

The dropdown uses the Fire Opal (dark) theme to match the merchant interface, with the trigger positioned in the top-right header area.

8. Interaction Behavior

Open/Close

  • ✓ Click trigger to toggle dropdown
  • ✓ Click outside to close
  • ✓ Escape key to close
  • ✓ Chevron rotates 180deg when open

Navigation Actions

  • ✓ Switch Merchant: Navigate to that merchant's dashboard
  • ✓ Back to Shopping: Redirect to my.saltwyk.com/passport
  • ✓ Go Live: Navigate to setup completion flow
  • ✓ Settings: Open settings page
  • ✓ Sign Out: Perform sign out, redirect to /login

10. Design Decisions

Why Top-Right?

Consistency with shopper experience. "Personal actions must always be in the same location" - top-right is where shoppers switch contexts, so merchants should do the same.

Why Fire Opal (Dark)?

The dropdown uses the dark theme to match the merchant sidebar and reinforce the "work" context. It creates visual cohesion within the merchant interface.

Why "Go Live" in Trial?

During trial, merchants haven't completed onboarding. The prominent "Go Live" CTA replaces the "Switch to" section to encourage setup completion.

Why No Avatar?

The trigger shows merchant name only (no avatar) to keep it compact. The dropdown provides full context with name, role, and optional merchant switching.

11. Usage Guidelines

Do

  • • Place in top-right of merchant header on all screens
  • • Show current merchant name in trigger
  • • Use Fire Opal (dark) theme for dropdown
  • • Include "Back to Shopping" for all users
  • • Show role badges for multi-merchant users
  • • Make dropdown right-aligned with trigger
  • • Use bottom sheet on mobile (<768px)
  • • Include keyboard navigation support

Don't

  • • Hide context switcher on any merchant screen
  • • Use Bright Opal (light) theme in merchant context
  • • Show user avatar in merchant dropdown
  • • Display exact role permissions in dropdown
  • • Left-align dropdown (always right-align)
  • • Show "Switch to" section for single-merchant users
  • • Allow dropdown to extend off-screen on mobile
  • • Omit "Back to Shopping" option

When to Show

  • • Always visible on all merchant screens (Dashboard, Transactions, Clearing Balance, Settings)
  • • Not shown on login, signup, or public pages
  • • Not shown during onboarding flow (until merchant has account)
  • • Positioned consistently in top-right, never hidden or moved

Which Variant to Use

User Scenario Variant Shows
Employee at one merchant Single Merchant Current context, Shopping, Settings, Sign Out
Consultant at 2+ merchants Multi-Merchant Current context, Switch to (other merchants), Shopping, Settings, Sign Out
Merchant in trial period Trial State Current context (Trial), Go Live CTA, Shopping, Settings, Sign Out
