Metric Card

Display key financial metrics, balances, and status information with clear visual hierarchy and semantic color coding.

Preview Mode

Overview

Metric Cards present key business data at a glance. They follow a consistent three-part structure: label (what the metric is), value (the data point), and subtext (contextual information). Use emerald for positive/available values only.

Purpose

Display financial metrics, balances, dates, and status

Location

Dashboard home, Clearing Balance, analytics views

Layout

Grid of 2-4 cards, responsive stacking on mobile

Design Decisions

Emerald for Positive Values Only: The emerald-600 color is reserved exclusively for positive financial values — money that's available, earned, or ready for use. Neutral metrics (pending amounts, dates, counts) use the default ink color to avoid false positive signals.

Color Semantics:

  • • Available Balance → emerald (ready to withdraw)
  • • Pending Settlement → ink (not yet available)
  • • Next Payout Date → ink (informational, not a value)
  • • Negative/Failed → magenta (error or loss)

Tabular Numbers: All financial values use font-variant-numeric: tabular-nums to ensure proper alignment when numbers change or are displayed in columns.

Usage Guidelines

Do

  • • Use emerald only for available/positive values
  • • Use tabular-nums for financial values
  • • Include helpful subtext context
  • • Keep labels short and scannable
  • • Stack cards on mobile for readability

Don't

  • • Don't use emerald for pending/neutral values
  • • Don't use proportional fonts for numbers
  • • Don't omit the label — values need context
  • • Don't use thread gradients in metric cards
  • • Don't exceed 4 metric cards in a row
