Component Specification

OAuth Connection

A reusable component for connecting external platforms via OAuth. Used in Settings → Apps for commerce platform integrations.

1. Overview

Purpose

The OAuth Connection component handles the complete lifecycle of connecting third-party platforms via OAuth 2.0. It manages all connection states from initial setup through ongoing maintenance.

8. OAuth Flow

State Transitions

Not Connected
    ↓ [User selects platform]
Connecting
    ↓ [OAuth callback success] → Connected
    ↓ [OAuth callback error]  → Error
    ↓ [User cancels]          → Not Connected

Connected
    ↓ [Token expires]         → Expired
    ↓ [User disconnects]      → Not Connected
    ↓ [Test connection fails] → Error

Error
    ↓ [User retries]          → Connecting
    ↓ [User changes platform] → Not Connected

Expired
    ↓ [User reconnects]       → Connecting
    ↓ [User disconnects]      → Not Connected

9. Design Decisions

Platform Icons: Large 48px icons with brand colors help users quickly identify their connected platform. The rounded corners (12px) match the card design language.

Status Badges: Pill-shaped badges with semantic colors provide instant visual feedback. Connected (emerald), Connecting (amber), Error (rose), Expired (orange).

Error Messaging: Errors use prominent rose-colored boxes with clear explanations and actionable next steps. Users should never be left wondering what went wrong.

Disconnect Action: Uses danger button styling (rose border) to indicate the destructive nature of disconnecting, requiring intentional action.
