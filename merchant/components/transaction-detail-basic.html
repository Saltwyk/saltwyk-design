5. Design Decisions

Why show only primary contributor in Transaction Tier?

Transaction Tier merchants need to know their sales are working but don't need full partnership intelligence. Showing the largest contributor provides transparency while keeping strategic audience overlap data as a premium feature. The "+ N others" indicator creates natural curiosity that drives upgrades to Partnership Tier.

Which merchant is the "primary contributor"?

The primary contributor is the merchant whose program contributed the most dollar value to the transaction. This is the most meaningful data point for understanding who drove the sale. We don't use alphabetical or random selection as those would be arbitrary and less informative.

Why remove contribution UUIDs?

Transaction Tier merchants don't need ledger-level audit trails. The UUIDs are primarily useful for Partnership Tier merchants who need to reconcile contributions with their own systems or debug specific point deductions. Removing them simplifies the UI and reinforces the value gap between tiers.

Why use "+ N others" instead of "N hidden"?

Language matters for perception. "Hidden" implies secrecy or restriction, creating negative feelings. "+ N others" is neutral and factual — it simply indicates there's more information available. This positive framing supports the upgrade path without making Transaction Tier feel punitive.

Why keep settlement details in Transaction Tier?

All merchants need visibility into fees and net amounts for financial planning. This isn't strategic intelligence — it's basic operational data. Hiding it would create frustration without driving meaningful upgrades.

6. Usage Guidelines

Do

  • • Show the largest contributor with their logo
  • • Use "+ N others" for additional contributors
  • • Include upgrade CTA in expanded view
  • • Keep settlement details visible
  • • Show wishlist badge (not full metadata)
  • • Use neutral, factual language

Don't

  • • Show logo stacks (single logo only)
  • • Display contribution UUIDs
  • • Say "hidden" or "locked"
  • • Show per-contributor breakdowns
  • • Include full wishlist metadata
  • • Make upgrade feel punitive

When to Use Each Variant

Transaction Tier (this page)

  • • Merchants who only process points
  • • Basic transaction reporting needs
  • • No partnership analytics required

Partnership Tier

  • • Merchants with partnership agreements
  • • Need full contributor visibility
  • • Require audience overlap insights
