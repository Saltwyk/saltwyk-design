13. Design Decisions

Horizontal Tabs: Tab navigation is horizontal rather than vertical to maximize content width for forms and data display. This follows common SaaS patterns where settings contain dense information.

Card-based Layout: Each subsection uses a card container to visually group related settings. This provides clear boundaries between different types of information and makes scanning easier.

Read-only Business Profile: Critical business information (name, address, EIN) is read-only to prevent accidental changes. Users must contact support for updates, ensuring compliance and audit trails.

URL-based Tab State: Each tab has its own URL (/settings/finance) for deep linking and browser history support. This allows bookmarking specific settings sections.

Emerald Primary Actions: Save and primary action buttons use emerald-600 to maintain consistency with the Saltwyk brand while providing clear visual hierarchy over secondary actions.

Two-Account Financial Architecture: The Finance section separates Settlement Account (Stripe-managed, read-only) from External Bank Account (merchant-owned, editable). This reflects the actual money flow: funds settle to Stripe first, then transfer to the merchant's bank. This separation provides FDIC protection on held funds and clear audit trails.

Auto-Fund with Clearing Balance Integration: Auto-fund settings use toggle + threshold pattern because merchants need granular control over when their clearing balance is replenished. The frequency estimate provides transparency about actual fund transfer cadence based on typical transaction volume.

Statement Cards with Color-Coded Amounts: Monthly statements use magenta-600 for fees (deductions) and emerald-600 for net amounts (positive) to provide at-a-glance financial health indicators. This color coding matches the Transactions screen for consistency.
