7. Design Decisions

Why a dark sidebar?

The dark intaglio sidebar creates visual separation between navigation and content, following the Fire Opal merchant treatment. It provides contrast that helps merchants focus on their data while maintaining brand cohesion.

Why show setup progress prominently?

Incomplete onboarding blocks payouts and full features. The prominent banner creates urgency and clear path to completion. Once setup is complete, the banner disappears and metrics take prominence.

Why limit transaction preview to 3-5 items?

The dashboard is an overview, not a detailed list. Showing recent activity provides a pulse check without overwhelming. The "View All" link provides clear access to the full transaction screen when needed.

Why use different layouts for mobile quick actions?

Desktop uses a grid because there's horizontal space and actions are secondary. Mobile uses a list format with chevrons because it's the expected mobile interaction pattern and provides clearer touch targets.
