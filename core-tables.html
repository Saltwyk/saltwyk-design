Tables

Data table patterns for displaying financial transactions, metrics, and structured information.

Basic Table

The foundational table style. Header row uses warm-100 background with uppercase labels. Body rows have white background with warm-200 bottom border.

Anatomy

  • • Header: bg-warm-100, uppercase labels, caption size (11px)
  • • Body rows: bg-white, border-b border-warm-200
  • • Cell padding: py-3 px-4 (12px 16px)
  • • Text alignment: Left for text, right for numbers

Status Cells

Status badges for table cells use thread-derived colors for semantic meaning.

Light Background

Cleared Success, complete, approved
Processing In progress, pending
Attention Needs review, expiring
Failed Error, rejected, critical
Info Informational, neutral

Dark Background (Fire Opal)

Cleared Success, complete
Processing In progress
Failed Error, rejected

Color Mapping

  • • Emerald: Success, cleared, approved, active
  • • Lime: Processing, in-progress, pending positive
  • • Orange: Warning, attention needed, expiring
  • • Magenta: Error, failed, rejected, critical
  • • Cyan: Informational, neutral, link-related

Responsive Behavior

Horizontal Scroll

Wrap table in overflow-x-auto container. Table maintains minimum column widths.

Priority Columns

Hide secondary columns on mobile using hidden md:table-cell. Keep ID, Amount, Status visible.

Card Transformation

On narrow viewports, transform rows into stacked cards with label-value pairs.
