Foundation Component

Tabs

Organize content into parallel views within the same context. Switching tabs is instant with no page reload.

01 — Philosophy

Parallel Organization

Tabs organize content into parallel views within the same context. Only one tab panel is visible at a time.

Instant Switching

Tabs should feel lightweight. Switching is instant with no page reload or heavy transitions.

Clear Indication

The active tab must be clearly distinguishable. Users should always know which view they're seeing.

02 — Anatomy

Tab List

Tab Trigger

Tab Indicator

Tab Panel

  • Tab list: Container for tab triggers (role="tablist")
  • Tab trigger: Clickable element for each tab (role="tab")
  • Tab indicator: Visual marker for active tab (underline, background)
  • Tab panel: Content area that changes based on selected tab

09 — Animation

Tab Indicator (Underline)

  • Behavior: Underline slides from previous tab to current tab
  • Duration: 200ms
  • Easing: ease-out

Tab Panel Transition

  • Option 1: Fade out old panel, fade in new panel (150ms)
  • Option 2: No animation (instant switch)
  • Recommendation: Instant switch for most cases

10 — Keyboard & Accessibility

Keyboard Navigation

  • Tab — Move focus to tab list, then to active panel
  • ← / → — Move between tabs (horizontal)
  • ↑ / ↓ — Move between tabs (vertical tabs)
  • Home — Move to first tab
  • End — Move to last tab
  • Enter / Space — Activate focused tab

ARIA Requirements

  • Tab list: role="tablist"
  • Each tab: role="tab"
  • Active tab: aria-selected="true"
  • Each panel: role="tabpanel"
  • Panel linked via: aria-labelledby
  • Hidden panels: hidden attribute
<!-- Accessible tab structure -->
<div role="tablist" aria-label="Settings tabs">
  <button role="tab" aria-selected="true" aria-controls="panel-1" id="tab-1">
    General
  </button>
  <button role="tab" aria-selected="false" aria-controls="panel-2" id="tab-2">
    Security
  </button>
</div>

<div role="tabpanel" id="panel-1" aria-labelledby="tab-1">
  <!-- Panel 1 content -->
</div>

<div role="tabpanel" id="panel-2" aria-labelledby="tab-2" hidden>
  <!-- Panel 2 content -->
</div>

13 — Implementation Checklist

Underline Tabs

  • Tab list: border-b warm-200 (light) / intaglio-700 (dark)
  • Active: emerald-500 underline (2px)
  • Default text: warm-600 (light) / warm-400 (dark)

Pill Tabs

  • Border radius: 6px
  • Active: emerald-100 bg (light) / emerald-500/20 (dark)
  • No tab list border

Boxed Tabs

  • Container: warm-100 bg (light) / intaglio-900 (dark), 4px padding
  • Active: white bg with shadow (light) / intaglio-700 (dark)

Vertical Tabs

  • Active: emerald-500 left border (3px)
  • Fixed width: 180-240px

Keyboard

  • Arrow keys navigate
  • Home/End jump to first/last
  • Enter/Space activate (if manual activation)

Accessibility

  • role="tablist" on container
  • role="tab" on triggers
  • role="tabpanel" on panels
  • aria-selected on active tab
