State Definitions

Core interactive states that apply across all components.

Default

The resting state of a component before any user interaction.

No modifier class

Hover

Visual feedback when the cursor hovers over an interactive element.

:hover

Active / Pressed

The moment of activation, typically while mouse is pressed down.

:active

Focused

Keyboard focus indicator, essential for accessibility.

:focus, :focus-visible

Disabled

Non-interactive state indicating unavailable action.

:disabled, [aria-disabled]

Loading

Processing state while waiting for a response.

.loading, [aria-busy]

Error

Validation failed or action resulted in an error.

[aria-invalid], .error

Success

Validation passed or action completed successfully.

.success, [aria-valid]

Empty

No content available, with guidance for next steps.

.empty-state

Accessibility

Essential requirements for accessible state communication.

Focus Indicators

  • • Never remove focus outlines without providing alternatives
  • • Focus rings must have 3:1 contrast against backgrounds
  • • Use :focus-visible to show focus only for keyboard navigation

ARIA Attributes

  • • Use aria-disabled="true" for disabled states
  • • Use aria-invalid="true" for error states
  • • Use aria-busy="true" for loading states
  • • Use aria-describedby to link error messages

Color Independence

  • • Never rely solely on color to communicate state
  • • Combine color with icons, text, or patterns
  • • Error states must include text explanation

Screen Reader Announcements

  • • Use aria-live="polite" for status updates
  • • Use aria-live="assertive" for errors
  • • Include loading state announcements
