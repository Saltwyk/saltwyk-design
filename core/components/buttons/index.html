Core Foundation

Button System

Buttons follow a 5-tier hierarchy based on visual weight. Primary demands attention; each subsequent tier recedes. Use the appropriate tier based on action importance.

Hierarchy Overview

The 5-Tier System

1. Primary

Main action. One per page ideally.

2. Secondary

Important alternative. "Learn More"

3. Tertiary

Soft/quiet actions.

4. Ghost

Minimal. "Cancel"

5. Destructive

Dangerous. "Delete"

On Dark Background

Tier 1

Primary Button

THE main action. Reserve for the single most important action you want users to take.

Accessibility

Focus States

All buttons must have visible focus indicators for keyboard navigation. Focus rings use indigo to distinguish from success states (emerald).

Guidelines

Do's and Don'ts

Do

  • One primary button per page (ideally)
  • Use secondary for important alternatives
  • Use tertiary for soft/quiet actions
  • Use ghost for cancel/dismiss
  • Reserve destructive for dangerous actions only
  • Use appropriate dark mode variants

Don't

  • Don't use multiple primaries competing
  • Don't use destructive for non-dangerous actions
  • Don't use emerald for non-interactive elements
  • Don't mix too many button types in one area
  • Don't use light mode buttons on dark backgrounds
  • Don't skip the hierarchy (e.g., tertiary as main CTA)
