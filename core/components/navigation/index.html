1. Philosophy

  • Navigation provides wayfinding — helping users understand where they are and how to move through the application
  • The sidebar is the primary navigation pattern for the merchant dashboard, providing persistent access to all major sections
  • Navigation should be stable and predictable — users rely on consistent placement and behavior
  • Keep navigation scannable with clear hierarchy and grouping

2. Navigation Types Overview

Different navigation patterns serve different use cases and surfaces.

Type Use Case Primary Surface
Sidebar Primary app navigation, persistent access to all major sections Merchant Dashboard
Top Navigation Marketing site navigation, simpler apps with fewer sections Marketing, Docs
Breadcrumbs Location indicator within page hierarchy, supports deep navigation All surfaces
Bottom Navigation Mobile primary navigation, thumb-accessible on smaller screens Shopper Passport (mobile)
Tabs In-page section switching, content organization within a view All surfaces →

3. Accessibility

ARIA requirements for each navigation type.

Sidebar Navigation

  • Landmark: role="navigation" with aria-label="Main"
  • Current page: aria-current="page"
  • Expandable sections: aria-expanded on trigger
  • Icons must have aria-hidden="true"
  • Skip link target at main content start

Top Navigation

  • Landmark: role="navigation" with aria-label="Primary"
  • Current page: aria-current="page"
  • Dropdown menus: aria-haspopup="menu"
  • Mobile menu toggle: aria-expanded, aria-controls
  • Logo link should describe destination

Breadcrumbs

  • Landmark: role="navigation" with aria-label="Breadcrumb"
  • Container: <ol> element for ordered list
  • Current page: aria-current="page" (last item)
  • Separators: decorative, use aria-hidden="true"

Bottom Navigation

  • Landmark: role="navigation" with aria-label="Main"
  • Current page: aria-current="page"
  • Icons: include visible label or aria-label
  • Badge counts: aria-label describing count
  • Touch target minimum: 44x44px

4. Implementation Checklist

Key specifications to verify during implementation.

Structure

  • ☐ Use semantic <nav> element
  • ☐ Include aria-label to distinguish nav regions
  • ☐ Mark current page with aria-current="page"
  • ☐ Use <ul>/<ol> for nav lists

Keyboard Navigation

  • ☐ All nav items reachable via Tab
  • ☐ Visible focus indicators (2px emerald ring)
  • ☐ Escape closes any open dropdowns
  • ☐ Skip link available for main content

Visual Design

  • ☐ Active state uses indigo-50 background
  • ☐ Hover state uses warm-50 background
  • ☐ 4.5:1 minimum contrast for text
  • ☐ Icons at 20px with 8px gap to label

Responsive Behavior

  • ☐ Sidebar collapses to icons on tablet
  • ☐ Sidebar becomes drawer on mobile
  • ☐ Touch targets minimum 44x44px
  • ☐ Bottom nav visible only on mobile
