Component Specification

Progress Indicators

Implementation specification for Saltwyk progress components. Communicate activity and duration to users.

Surface-specific documentation: Merchant Setup Progress covers compact dot-based progress for onboarding flows, including mobile header patterns and step cards.

1. Philosophy

Progress indicators communicate that something is happening and how far along it is. They reduce perceived wait time and set expectations. Determinate progress (known duration) is always preferred over indeterminate.

Use progress bars for uploads and multi-step processes. Use spinners for brief, unknown-duration operations. Use skeletons to show content layout while loading.

2. Progress Types

Type Use Case
Progress Bar File uploads, multi-step processes, known duration
Circular Progress Loading states, compact spaces
Spinner Indeterminate loading, button states
Skeleton Content loading placeholders
Steps/Stepper Multi-step wizards, onboarding flows

Progress Bar

Spinner

@keyframes spin { from { transform: rotate(0deg); } to { transform: rotate(360deg); } } .spinner { border-radius: 9999px; border: 2px solid hsl(150 100% 27%); border-top-color: transparent; animation: spin 750ms linear infinite; } .spinner-sm { width: 16px; height: 16px; } .spinner-md { width: 24px; height: 24px; } .spinner-lg { width: 32px; height: 32px; }

11. Implementation Checklist

Progress Bar

  • Heights: 4px, 8px, 12px
  • Track: warm-200 (light) / intaglio-700 (dark)
  • Fill: emerald, orange (warning), magenta (error)
  • Animation: 300ms width transition

Spinner

  • Sizes: 12px, 16px, 24px, 32px, 48px
  • Color: emerald-500 default, white on buttons
  • Animation: 750ms rotation

Skeleton

  • Background: warm-100 / intaglio-700
  • Shimmer animation: 1.5s
  • Common shapes: text, avatar, card

Stepper

  • Indicator size: 32px default
  • States: completed, current, upcoming, error
  • Completed: emerald bg with checkmark
  • Connector colors match state
