Component Specification

Buttons

Implementation specification for Saltwyk button components. Every state, variant, and pixel value defined.

1. Philosophy

Buttons are the primary call-to-action element in the Saltwyk design system. They should feel substantial and trustworthy, reflecting the currency aesthetic that defines our visual language.

Primary buttons use emerald to draw attention and signal action. Secondary buttons recede to support hierarchy. On dark surfaces (Fire Opal treatment), buttons gain subtle glow effects that reinforce the premium, authoritative feel.

2. Anatomy

Icon (optional)
Container (bg, border, radius)

Label (text)

Container

Background, border, border-radius. Defines the button's shape and fill.

Label

Button text. Always Inter medium weight, centered.

Icon

Optional. Can be leading (left) or trailing (right). Lucide icons.

Focus Ring

2px emerald ring with 2px offset. Visible on keyboard focus.

4. Button Hierarchy & Usage

When to use each variant. Understanding hierarchy helps create clear, usable interfaces.

Maximum: ONE per view

Use for: The single most important action on the page.

Examples: "Complete Purchase", "Save Changes", "Get Started"

Never: Secondary actions, cancel buttons, navigation

Use for: Supporting actions alongside Primary.

Examples: "Save Draft" next to "Publish", "Back" next to "Continue"

Common pairing: Primary + Secondary in forms and modals

Dark mode only

Use for: Alternative to Secondary on dark surfaces when emerald isn't appropriate.

Examples: "Learn More" on a dark hero, neutral actions on dark cards

Character: Visible but doesn't carry brand color weight

Note: Only available on dark surfaces—use Secondary or Outline on light

Use for: Alternative to Secondary with lighter visual weight.

Examples: Filter toggles, optional actions, less emphasis than Secondary

Choose over Secondary: When you want the action to recede more

Use sparingly

Use for: Celebratory or highlighted moments.

Examples: "Claim Reward", "Connect Program", special promotions

Caution: Thread gradient draws significant attention

Use for: Tertiary actions, navigation, actions within content.

Examples: "Cancel", "Close", table row actions, icon-only actions

Character: Should almost disappear until needed

Dangerous actions ONLY

Use for: Irreversible or dangerous actions.

Examples: "Delete Account", "Remove Program", "Cancel Subscription"

Never: Regular cancel buttons (use Ghost instead)

Always: Require confirmation for destructive actions

7. Thread Accent Variant (Special)

For moments of celebration or emphasis, buttons can include a thread gradient border.

Outline with Thread Border

Thread gradient borders are only appropriate on Outline variants. On filled buttons, the light thread colors create contrast collision with the background.

On hover, fill transitions to gradient-cool-bg (8% saturation).

Implementation Note

Thread borders require a wrapper approach: outer element has the gradient background with padding matching border width, inner span has the button styling. Border-radius on both must account for the padding offset (outer: 6px, inner: 4px).

12. Implementation Checklist

Verification checklist for developers refactoring the component.

Base Structure

  • ☐ Border radius is exactly 6px (rounded-md)
  • ☐ Border width is 1.5px
  • ☐ Font weight is 500 (medium)
  • ☐ Transition is 150ms ease-out on all properties
  • ☐ Focus ring is 2px emerald-500 with 2px offset (ALL variants including Destructive)
  • ☐ Icon gap is 8px

Sizes

  • ☐ Small: 32px height, 12px horizontal padding, 13px font
  • ☐ Default: 40px height, 20px horizontal padding, 14px font
  • ☐ Large: 48px height, 28px horizontal padding, 16px font
  • ☐ Icon-only buttons are square (width = height)

Primary Variant

  • ☐ Default: emerald-500 background, white text
  • ☐ Hover: emerald-600 background, subtle shadow
  • ☐ Active: emerald-700 background, no shadow
  • ☐ Disabled: 50% opacity

Secondary Variant

  • ☐ Default: warm-100 background, warm-300 border, ink text
  • ☐ Hover: warm-200 background, warm-400 border
  • ☐ Disabled: 50% opacity on all

Dark Mode (Fire Opal)

  • ☐ Primary gets subtle glow on dark backgrounds
  • ☐ Secondary (Emerald Ghost): 10% bg, 30% border, emerald-400 text
  • ☐ Secondary hover: 20% bg, 40% border, emerald-300 text
  • ☐ Tertiary (Inverted): warm-100 bg, intaglio-900 text
  • ☐ Tertiary hover: white bg (dark surfaces only)
  • ☐ Ghost and Outline adapt to dark surface
  • ☐ Destructive unchanged (magenta works on dark)

Accessibility

  • ☐ Focus ring visible on all variants
  • ☐ Disabled buttons have aria-disabled="true"
  • ☐ Loading buttons have aria-busy="true"
  • ☐ Icon-only buttons have aria-label

13. Code Reference

Copy-pasteable CSS/Tailwind classes for each variant.

Primary

/* Base */
.btn-primary {
  @apply bg-emerald-500 text-white font-medium rounded-md;
  @apply border-[1.5px] border-transparent;
  @apply transition-all duration-150 ease-out;
  @apply focus-visible:ring-2 focus-visible:ring-emerald-500 focus-visible:ring-offset-2;

/* Hover */

  @apply bg-emerald-600;

.btn-primary:active {
  @apply bg-emerald-700;
  box-shadow: none;
